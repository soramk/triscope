# 実装計画: モバイル視認性と操作性の向上

## 目的

スマートフォン等のモバイルデバイスにおいて、Triscopeのコンテンツが適切に表示され、快適に操作できるように改善します。

## 主要な変更点

### 1. レスポンシブ・サイドバーの実装 (`index.html`, `style.css`)

- **ハンバーガーボタンの追加**: モバイル表示（768px以下）で表示されるメニューボタンを追加。
- **ドロワーメニュー**: サイドバーをデフォルトで非表示にし、ボタン押下で左からスライド表示する形式に変更。
- **オーバーレイ**: メニュー表示時にコンテンツを暗くするオーバーレイを追加。

### 2. レイアウトとタイポグラフィの最適化 (`style.css`)

- **余白の調整**:
  - `padding: 2rem 4rem` をモバイルでは `padding: 1rem` 程度に縮小。
  - セクション間のマージンを最適化。
- **フォントサイズの動的調整**:
  - `h1` を `2.5rem` -> `1.8rem`
  - `h2` を `1.8rem` -> `1.4rem`
  - 本文の読みやすさを確保。
- **グリッドの調整**: `portal-grid` のカラム数を画面幅に合わせて最適化。

### 3. モバイルフレンドリーな UI 要素

- **タッチターゲット**: すべてのナビゲーションリンクとボタンの最小高さを 44px に設定。
- **iframe の最適化**: iframe内のスクロールがスムーズになるよう CSS を調整。

## 手順

1. `index.html` にメニューボタンとオーバーレイの HTML 構造を追加。
2. `style.css` にモバイル用のメディアクエリを追加。
3. `index.html` にメニュー開閉の JavaScript ロジックを追加。
4. `home.html` やその他の主要ページの表示を確認し、必要に応じて微調整。

## 完了後の確認事項

- [ ] 768px以下の画面幅でサイドバーが隠れているか。
- [ ] ハンバーガーメニューでサイドバーが開閉できるか。
- [ ] コンテンツのパディングが適切で、横スクロールが発生していないか。
- [ ] 文字が小さすぎたり、見出しが大きすぎたりしないか。
